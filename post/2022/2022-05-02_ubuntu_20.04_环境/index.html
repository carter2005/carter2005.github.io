ubuntu 20.04.4开发环境

2022-05-02
Linux
次阅读

2022年注定是一个不平凡的一年，俄罗斯，乌克兰的战争离我们很远，近在眼前的上海疫情，先是3月初在公司隔离4天，回来后就在家办公，已经五一节了，依然形势不容乐观。所幸我们的工作远程办公很容易，工作倒也没有什么大影响，每天还不用浪费路上的3个小时，家里可以用自己的电脑，醒了就工作，困了就休息，其实比去公司更方便。

本篇主要想谈谈现在的pc操作系统市场，以及近期部署的ubuntu 20.04开发环境。截至目前，主流的操作系统就下面几个：

  • win7：特别经典好用的一个系统，无奈现在的新硬件基本都抛弃了对它的支持，只能在虚拟机里面苟延残喘了。
  • win10：经历了多年无穷无尽的更新，基本也到了生命末期，好用的还是LTSC 2019，关掉更新用起来还是省心的，综合看来，依然是目前最好用的系统。
  • win11：接过了bug10的交接棒，只有最新的硬件才能安装。不发展上5年是不能当作主力系统使用的。
  • Linux：自从一大波发行版把界面换成了gnome3，感觉Linux就丧失了灵魂，失去了方向，很多时候纯粹为了更新而更新，夹在windows，macos中间不知所措，基本一个能打的图形界面都没有了，比windows，macos落后了一大截。
  • macos：Apple的步伐一直很激进，一直按着自己的想法在改造系统，虽然现在印度人的审美水准导致UI美观度下降，但依然是最漂亮的，Apple silicon也确实引导了潮流，开创了硬件研发的新河。

工作中开发android那些年用的是ubuntu 14.04.6，用起来非常的省心，开发AI加速卡的时候，用的ubuntu 16.04.7，这个系统在1080p显示器上鼠标有些飘，加速度乱跑，在公司的2k显示器上问题比较不明显，体验尚可。后期发布的18.04，20.04，22.04，更换成了gnome界面，简直一代更比一代差，最近的项目目标环境是ubuntu 20.04，新系统对新硬件的支持要好上不少，是时候更新主力工作环境了。

ubuntu 20.04截止目前已经迭代了5个小版本到了22.04.4，稳定性不容置疑，安装没什么好讲的，重点是安装以后的定制。

root登陆

因为经常要和硬件打交道，老得sudo切换root，我平时还是更喜欢直接使用root账号

  1. 以普通用户登录系统，创建root用户的密码

在终端输入命令：sudo passwd root

然后输入设置的密码，输入两次，这样就完成了设置root用户密码了（注意Linux系统下密码是没有回显的）
  1. 修改 50-ubuntu.conf 文件

在终端输入命令：sudo gedit /usr/share/lightdm/lightdm.conf.d/50-ubuntu.conf

在文件末尾增加如下两行：
greeter-show-manual-login=true
all-guest=false

前面加 # 注释掉第三行的 auth required pam_succeed_if.so user != root quiet_success
